TOPIC: Event Driven Programming
LEVEL: Level 2 IT – Unit 5: Introduction to Programming (Pearson BTEC – Jordan)

LEARNING_OUTCOMES:
- فهم نموذج البرمجة القائم على الأحداث (Event Driven Programming).
- التعرف على المفاهيم الأساسية مثل Event و Event Handler.
- ربط النموذج بأمثلة بسيطة من التطبيقات اليومية.
- معرفة المزايا والعيوب بمستوى Level 2.

CORE_EXPLANATION:
البرمجة القائمة على الأحداث (Event Driven Programming) هي أسلوب برمجة يعتمد على حدوث حدث (Event) لتنفيذ جزء معين من البرنامج.

الحدث (Event) يمكن أن يكون:
- الضغط على زر (Button Click)
- الضغط على مفتاح من لوحة المفاتيح (Key Press)
- لمس الشاشة (Touch)
- تحريك الفأرة (Mouse Movement)

في هذا النموذج، يبقى البرنامج يعمل من خلال حلقة تُسمّى حلقة الأحداث (Event Loop).
عند حدوث الحدث، يتم تشغيل جزء من الكود يُسمّى معالج الحدث (Event Handler).

العناصر الأساسية:
- Event: الشيء الذي يحدث.
- Event Handler: الكود الذي يعمل عند حدوث الحدث.
- Event Loop: تبقي البرنامج يعمل وتستقبل الأحداث.

هذا النموذج شائع في تطبيقات الواجهات الرسومية (GUI) وتطبيقات الهواتف.

PROGRAMMING_LANGUAGES:
يمكن تطبيق Event Driven Programming باستخدام لغات مثل:
- Python
- JavaScript
- Java

SIMPLE_CODE_EXAMPLE:
Pseudo example:
When a button is clicked:
    print("Button was clicked")

ADVANTAGES:
- سهل التعلم للمبتدئين.
- يعتمد على تفاعل المستخدم.
- سهل إضافة ميزات جديدة.

DISADVANTAGES:
- تدفق التنفيذ قد يكون أصعب في البرامج الكبيرة.
- يحتاج إلى تنظيم جيد للأحداث.

FACTS_FOR_QUIZZES:
- Event Driven Programming تعتمد على Event.
- Event Handler ينفذ الكود عند حدوث Event.
- Event Loop تبقي البرنامج يعمل.
