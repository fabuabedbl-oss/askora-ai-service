TOPIC: Procedural Programming
LEVEL: Level 2 IT – Unit 5: Introduction to Programming (Pearson BTEC – Jordan)

LEARNING_OUTCOMES:
- فهم مفهوم البرمجة الإجرائية.
- التعرف على فكرة الأوامر والإجراءات.
- مقارنة البرمجة الإجرائية مع البرمجة كائنية التوجه.
- معرفة المزايا والعيوب.

SCENARIO:
يريد المبرمج كتابة برنامج بسيط.
البرنامج يقرأ رقم من المستخدم.
ثم يعرض نتيجة معينة.
البرنامج يعمل من البداية إلى النهاية بخطوات واضحة.

SCENARIO_SOLUTION:
يتم تنفيذ الأوامر بالترتيب.
كل أمر يتم تنفيذه بعد الذي قبله.
يمكن استخدام إجراءات أو دوال لتقسيم البرنامج وتنظيمه.
البرنامج ينتهي بعد تنفيذ آخر أمر.

CORE_EXPLANATION:
البرمجة الإجرائية (Procedural Programming) هي أسلوب برمجة يعتمد على تنفيذ مجموعة من الأوامر خطوة بخطوة.
البرنامج يبدأ من أول سطر وينفذ الأوامر بالتسلسل حتى النهاية.
يتم استخدام إجراءات أو دوال لتنظيم الكود.

العناصر الأساسية:
- Instructions: أوامر يتم تنفيذها.
- Procedure: مجموعة أوامر تؤدي مهمة معينة.
- Function: إجراء يعيد قيمة.

PROGRAMMING_LANGUAGES:
يمكن تطبيق Procedural Programming باستخدام لغات مثل:
- C
- Python
- Pascal

PSEUDO_CODE_EXAMPLE:
Start program
Read number
Print result
End program

ADVANTAGES:
- سهل التعلم للمبتدئين.
- واضح وبسيط.
- مناسب للبرامج الصغيرة.

DISADVANTAGES:
- صعب التوسعة في البرامج الكبيرة.
- إعادة استخدام الكود أقل مقارنة بـ OOP.

FACTS_FOR_QUIZZES:
- Procedural Programming تعتمد على الأوامر.
- تنفيذ الكود يتم بالتسلسل.
- Procedure هو مجموعة أوامر.
