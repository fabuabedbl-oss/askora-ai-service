TOPIC: Procedural Programming
LEVEL: Level 2 IT – Unit 5: Introduction to Programming (Pearson BTEC – Jordan)

LEARNING_OUTCOMES:
- فهم مفهوم البرمجة الإجرائية.
- تنفيذ الأوامر بشكل متسلسل.
- استخدام Instructions و Procedures و Functions.
- التمييز بين المتغيرات المحلية والعالمية.
- فهم أسلوب التنفيذ المتسلسل (Top-down).
- معرفة المزايا والعيوب.
- مقارنة البرمجة الإجرائية مع OOP.

CORE_EXPLANATION:
البرمجة الإجرائية (Procedural Programming) هي أسلوب برمجة يعتمد على تنفيذ مجموعة من الأوامر خطوة بخطوة.
يبدأ البرنامج من أول سطر وينفذ التعليمات بالتسلسل حتى النهاية.
يمكن تقسيم البرنامج إلى Procedures أو Functions لتنظيم الكود.

العناصر الأساسية:
- Instruction: أمر يتم تنفيذه.
- Procedure: مجموعة أوامر تؤدي مهمة معينة.
- Function: إجراء يعيد قيمة.
- Local Variables: متغيرات تُستخدم داخل الإجراء فقط.
- Global Variables: متغيرات يمكن استخدامها في كامل البرنامج.

EXECUTION_MODEL:
تعتمد البرمجة الإجرائية على أسلوب التنفيذ من أعلى إلى أسفل (Top-down)، حيث يتم تنفيذ الأوامر بالتسلسل.

SCENARIO:
يريد المبرمج كتابة برنامج بسيط.
البرنامج يقرأ رقمًا من المستخدم.
ثم يعرض نتيجة.
ينتهي البرنامج بعد تنفيذ آخر أمر.

PSEUDO_CODE_EXAMPLE:
Start program
Read number
Print result
End program

ADVANTAGES:
- سهل التعلم للمبتدئين.
- واضح وبسيط.
- يتطلب ذاكرة أقل.
- مناسب للبرامج الصغيرة.

DISADVANTAGES:
- صعب التوسعة في البرامج الكبيرة.
- إعادة استخدام الكود أقل مقارنة بـ OOP.
- صيانة البرنامج تصبح أصعب مع الوقت.

FACTS_FOR_QUIZZES:
- Procedural Programming تعتمد على الأوامر.
- تنفيذ الكود يتم بالتسلسل.
- Function تعيد قيمة.
- Local Variables تختلف عن Global Variables.
