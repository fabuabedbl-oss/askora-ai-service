TOPIC: Object-Oriented Programming (OOP)
LEVEL: Level 2 IT – Unit 5: Introduction to Programming (Pearson BTEC – Jordan)

LEARNING_OUTCOMES:
- فهم مفهوم البرمجة كائنية التوجه.
- التعرف على Class و Object.
- فهم Attributes و Methods.
- فهم Encapsulation و Inheritance و Polymorphism.
- فهم أسلوب التنفيذ من أسفل إلى أعلى (Bottom-up).
- معرفة المزايا والعيوب.
- التمييز بين OOP والبرمجة الإجرائية.

CORE_EXPLANATION:
البرمجة كائنية التوجه (Object-Oriented Programming) هي أسلوب برمجة يعتمد على تمثيل البرنامج باستخدام كائنات (Objects).
كل كائن يتم إنشاؤه من قالب يسمى Class.
الـ Class يحتوي على خصائص (Attributes) تصف الكائن، ووظائف (Methods) تحدد ما يمكن للكائن فعله.

العناصر الأساسية:
- Class: قالب لإنشاء الكائنات.
- Object: نسخة من الـ Class.
- Attribute: خاصية تصف الكائن.
- Method: وظيفة يقوم بها الكائن.

المفاهيم المتقدمة:
- Encapsulation: إخفاء تفاصيل الكائن الداخلية.
- Inheritance: وراثة الخصائص والوظائف من Class آخر.
- Polymorphism: استخدام نفس العملية بطرق مختلفة.

EXECUTION_MODEL:
تعتمد OOP على أسلوب التنفيذ من أسفل إلى أعلى (Bottom-up)، حيث يتم إنشاء كائنات صغيرة ثم تجميعها لبناء النظام الكامل.

SCENARIO:
يريد المبرمج إنشاء برنامج لإدارة سيارات.
كل سيارة لها لون وسرعة.
بدل كتابة كود منفصل لكل سيارة، يتم إنشاء Class واحد.
يتم إنشاء عدة Objects من هذا الـ Class.

PSEUDO_CODE_EXAMPLE:
Class Car
    color
    speed
    drive()

Create object myCar from Car

ADVANTAGES:
- تنظيم أفضل للكود.
- سهولة إعادة استخدام الكود.
- مناسب للبرامج الكبيرة.
- سهولة الصيانة والتطوير.

DISADVANTAGES:
- يحتاج إلى فهم أعمق للمفاهيم.
- قد يكون معقدًا للمبتدئين.
- يتطلب مهارات أعلى مقارنة بالبرمجة الإجرائية.

FACTS_FOR_QUIZZES:
- Object يتم إنشاؤه من Class.
- Encapsulation تعني إخفاء التفاصيل.
- Inheritance تسمح بإعادة استخدام الكود.
- OOP تعتمد على الكائنات.
