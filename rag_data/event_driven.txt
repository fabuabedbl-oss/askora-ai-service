TOPIC: Event-Driven Programming
LEVEL: Level 2 IT – Unit 5: Introduction to Programming (Pearson BTEC – Jordan)

LEARNING_OUTCOMES:
- فهم نموذج البرمجة القائم على الأحداث.
- التعرف على Event و Event Handler و Event Listener.
- فهم دور Event Loop في تنفيذ البرنامج.
- فهم أسلوب التنفيذ من أعلى إلى أسفل (Top-down).
- ربط البرمجة القائمة على الأحداث بتفاعل المستخدم.
- معرفة المزايا والعيوب.

CORE_EXPLANATION:
البرمجة القائمة على الأحداث (Event-Driven Programming) هي أسلوب برمجة يعتمد على انتظار حدوث حدث لتنفيذ جزء معين من البرنامج.
البرنامج لا يعمل بشكل متسلسل من البداية إلى النهاية، بل يبقى في حالة انتظار حتى يحدث تفاعل من المستخدم أو النظام.

العناصر الأساسية:
- Event: شيء يحدث مثل الضغط على زر أو إدخال بيانات.
- Event Handler: الكود الذي يتم تنفيذه عند حدوث الحدث.
- Event Listener: أداة تراقب حدوث الحدث وتبلّغ البرنامج عند وقوعه.
- Event Loop: آلية تبقي البرنامج يعمل وتنتظر الأحداث باستمرار.

EXECUTION_MODEL:
يستخدم هذا النموذج أسلوب التنفيذ من أعلى إلى أسفل (Top-down)، حيث يتم إعداد البرنامج أولاً ثم ينتظر الأحداث لتنفيذ الأجزاء المناسبة عند حدوثها.

SCENARIO:
يقوم المستخدم بفتح تطبيق.
التطبيق يعرض زرًا على الشاشة.
المستخدم يضغط على الزر.
يُعتبر الضغط حدثًا.
يقوم Event Listener بالتقاط الحدث.
ينفّذ Event Handler الكود المرتبط بالحدث وتظهر رسالة للمستخدم.

PSEUDO_CODE_EXAMPLE:
Program is running
Wait for event

When button is clicked
    Execute event handler
    Display message

ADVANTAGES:
- تجعل البرنامج تفاعليًا.
- سهلة الفهم للمبتدئين.
- مناسبة للتطبيقات الحديثة.
- تسمح بإضافة وظائف جديدة بسهولة.

DISADVANTAGES:
- صعوبة تتبع تدفق التنفيذ في البرامج الكبيرة.
- قد تؤثر على موارد النظام.
- تحتاج إلى تنظيم جيد للأحداث.

FACTS_FOR_QUIZZES:
- Event Handler يعمل عند حدوث Event.
- Event Listener يراقب الحدث.
- Event Loop تبقي البرنامج في حالة انتظار.
