TOPIC: Event-Driven Programming
LEVEL: Level 2 IT – Unit 5: Introduction to Programming (Pearson BTEC – Jordan)

LEARNING_OUTCOMES:
- فهم نموذج البرمجة القائم على الأحداث.
- التعرف على Event و Event Handler.
- ربط النموذج بتفاعل المستخدم.
- معرفة المزايا والعيوب.

SCENARIO:
يقوم المستخدم بفتح تطبيق.
التطبيق يعرض زر على الشاشة.
المستخدم يضغط على الزر.
بعد الضغط، تظهر رسالة للمستخدم.

SCENARIO_SOLUTION:
البرنامج لا يعرف متى سيضغط المستخدم على الزر.
لذلك يبقى البرنامج يعمل وينتظر.
عند ضغط المستخدم على الزر، يعتبر هذا حدث.
البرنامج يلتقط الحدث.
ثم ينفذ الكود المرتبط بهذا الحدث ويعرض رسالة.

CORE_EXPLANATION:
البرمجة القائمة على الأحداث (Event Driven Programming) هي أسلوب برمجة يعتمد على انتظار حدوث حدث لتنفيذ جزء معين من البرنامج.
البرنامج لا يعمل بشكل متسلسل من البداية إلى النهاية، بل ينتظر تفاعل المستخدم أو النظام.

العناصر الأساسية:
- Event: شيء يحدث مثل الضغط على زر.
- Event Handler: الكود الذي يعمل عند حدوث الحدث.
- Event Loop: تبقي البرنامج يعمل وتنتظر الأحداث.

PROGRAMMING_LANGUAGES:
يمكن تطبيق Event Driven Programming باستخدام لغات مثل:
- Python
- JavaScript
- Java

PSEUDO_CODE_EXAMPLE:
Program is running
Wait for event

When button is clicked
    Display message

ADVANTAGES:
- تجعل البرنامج تفاعلياً.
- سهلة الفهم للمبتدئين.
- مناسبة للتطبيقات الحديثة.

DISADVANTAGES:
- صعوبة تتبع التنفيذ في البرامج الكبيرة.
- تحتاج إلى تنظيم جيد للأحداث.

FACTS_FOR_QUIZZES:
- Event Handler يعمل عند حدوث Event.
- Event Loop تبقي البرنامج يعمل.
